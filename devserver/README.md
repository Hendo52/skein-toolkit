# Dev Server Scripts

Rent a cloud GPU on [Vast.ai](https://vast.ai), pull AI models, and connect VS Code's [Continue.dev](https://continue.dev) chat — with one command.

## Prerequisites

- **Windows** with PowerShell 7+
- **Python 3.10+** with pip (from [python.org](https://python.org))
- **VS Code** with the [Continue.dev extension](https://marketplace.visualstudio.com/items?itemName=Continue.continue)
- A **[Vast.ai account](https://vast.ai)** with billing configured

---

## Quick start (first time)

### 1. Clone the repo

```bash
git clone https://github.com/yourname/your-repo.git
cd your-repo
```

### 2. Run the setup wizard

In VS Code: open the Command Palette, run **Tasks: Run Task → Dev Server: First-Time Setup**.

Or directly in a terminal:

```powershell
scripts\setup-devserver.ps1
```

The wizard will:
- Install the `vastai` CLI (via pip) if it's not already installed
- Ask for your Vast.ai API key (get it from [cloud.vast.ai/account](https://cloud.vast.ai/account))
- Generate an SSH key pair (or let you point to an existing one)
- Upload the public key to your Vast.ai account
- Ask for your git repo URL (used to clone your code onto each fresh instance)
- Write your personal settings to `~/.devserver-config.ps1` (not tracked in git)

### 3. Connect

In VS Code: **Tasks: Run Task → Dev Server**.

Or:

```powershell
scripts\rent-devserver.ps1
```

Pick a use case when prompted. The script does the rest.

---

## Daily workflow

```
Run "Dev Server" task
  → pick use case
  → script rents GPU (or reconnects to existing stopped instance)
  → wait ~2 min for boot
  → Ollama starts automatically, required models pull (first time: 5-30 min)
  → Continue.dev chat panel opens in VS Code
  → work...
  → Ctrl+C (or close the terminal) to close the tunnel
  → instance stays alive but idle — no GPU charges, disk only
```

To stop billing entirely:

```powershell
scripts\stop-devserver.ps1
```

## Use cases

| Use case | Models | Min VRAM | Max price |
|---|---|---|---|
| `chat` (default) | llama3.3:70b + qwen2.5-coder:7b | 48 GB | $0.90/hr |
| `batch` | deepseek-r1:32b | 20 GB | $0.60/hr |
| `completions` | qwen2.5-coder:7b | 12 GB | $0.30/hr |

The script selects the cheapest Vast.ai offer that meets the VRAM requirement and stays within the price ceiling.

**chat** is the best starting point. It gives you a large language model for conversation and a fast 7B model for inline code completions.

## Safety watcher

A safety watcher opens automatically in a separate terminal when you connect. It monitors Ollama activity and stops the instance when any limit is reached.

| Condition | Default | Config variable |
|---|---|---|
| Ollama idle (no loaded model) | 30 minutes | `$WatchIdleMinutes` |
| Session hard cap | 8 hours | `$WatchMaxSessionHours` |
| Spend hard cap | $8.00 | `$WatchMaxSpendUsd` |

Vast.ai auto-tops-up your credit without confirmation. This watcher is the circuit breaker that prevents runaway charges.

## Config reference

Personal settings live in `~/.devserver-config.ps1`, generated by `setup-devserver.ps1`. This file is never tracked in git.

Repo-level defaults live in `devserver/devserver.config.ps1`. Do not edit this file directly — put your overrides in the user config.

| Variable | Default | Description |
|---|---|---|
| `$DevServerKey` | `~\.ssh\vast_key` | Path to your SSH private key |
| `$DevServerRepo` | `/root/repo` | Remote path where the repo lives on the instance |
| `$DevServerRepoUrl` | _(empty)_ | Git URL cloned onto fresh instances (empty = skip) |
| `$WatchIdleMinutes` | `30` | Stop after this many minutes of Ollama idle |
| `$WatchMaxSessionHours` | `8` | Hard session cap in hours |
| `$WatchMaxSpendUsd` | `8.00` | Hard spend cap in USD |
| `$BatchFilePath` | `$env:SKEIN_BATCH_FILE` | Tik/tok batch file path. Set `SKEIN_BATCH_FILE` env-var to your project's batch file. Empty = batch-aware mode disabled. |
| `$DevServerMcpTunnelPort` | `3101` | Local port for the GPU remote MCP server tunnel. 3101 avoids conflicting with the Skein local MCP server on 3100. |

To change a setting, either re-run `setup-devserver.ps1` or edit `~/.devserver-config.ps1` directly.

## MCP server endpoints

When the GPU devserver is active, two MCP servers are available:

| Endpoint | Port | What it is |
|---|---|---|
| `http://localhost:3100/sse` | 3100 | **Skein local MCP** — always-on, tools for filesystem, search, skills |
| `http://localhost:3101/sse` | 3101 | **GPU remote MCP** — tunneled from `devserver-mcp.py` on the remote instance; tools for run_shell, read/write on the GPU machine |

Register both in your coding agent (Odysseus, aider, Continue.dev) to get local tools from Skein and remote shell access from the GPU in the same session.

## What happens under the hood

1. Queries Vast.ai for the cheapest offer within the VRAM and price requirements.
2. Creates the instance (or resumes a stopped one if one exists).
3. Waits for `actual_status == running` (up to 30 minutes).
4. Clones your repo onto the instance (if `$DevServerRepoUrl` is set).
5. Writes `~/.ssh/config` with a `vast-devserver` alias for the current IP/port.
6. Starts Ollama on the instance if it isn't already running.
7. Pulls any missing models for the chosen use case.
8. Patches `~/.continue/config.yaml` to use `localhost:11435` (restored when the tunnel closes).
9. Opens the Continue.dev panel in the current VS Code window.
10. Starts the safety watcher in a new terminal.
11. Opens a blocking SSH tunnel: `localhost:11435 → remote:11434`.

## Troubleshooting

**"vastai not found"**
Run `pip install vastai`, then check that Python's Scripts folder is on your PATH. Re-run `setup-devserver.ps1` if needed.

**"Local port 11435 is in use"**
A previous tunnel didn't close cleanly. Find and kill it:
```powershell
(Get-NetTCPConnection -LocalPort 11435 -ErrorAction SilentlyContinue).OwningProcess | ForEach-Object { Stop-Process -Id $_ -Force }
```
Then re-run `rent-devserver.ps1`.

**"API key / 401 error"**
Re-run `setup-devserver.ps1` to re-enter your key, or run:
```powershell
vastai set api-key <your-key>
```

**"Vast.ai requires 2FA"**
The script handles this automatically — it sends the email code and prompts you to enter it.

**"Instance not running after 30 minutes"**
Vast.ai provisioning occasionally times out. Check [cloud.vast.ai/instances](https://cloud.vast.ai/instances). Re-run `rent-devserver.ps1` to reconnect once the instance shows as running.

**Continue.dev shows no models / connection refused**
The tunnel may not be open. Re-run `rent-devserver.ps1`. While the tunnel is active, `~/.continue/config.yaml` should show `localhost:11435/v1`. If it shows `localhost:11434`, a previous session didn't restore the config — edit it manually or re-run.

**Model pull is slow**
70B models are 40+ GB. First pull on a fresh instance takes 10–30 minutes depending on bandwidth. Subsequent runs (if the disk is preserved) skip the pull entirely.

**Instance stopped but I didn't close the tunnel**
The safety watcher stopped it due to an idle/time/spend limit. Increase the caps in `~/.devserver-config.ps1` and re-run.
